課題3 提出一式（Table 2・Table 3・Table 4 の再現）
====================================================


ようやくプログラム書き終わりました，こっちもほとんどAIで作ったんですが
初心者っぽく間違えようと思ったんだけど，AIに最後見直しするようにしたっていうふうなレポートにした方が良くて，それならAIがそのミスにも気づくから結局ミスなく完璧な表ができるようにしています．


このフォルダは、課題3（論文の Table 2・3・4 をRで再現するやつ）の一式です。
どのファイルが何なのか、どう使えばいいのかをここに書いておきます。




■ フォルダの中身と使い方

・hl_qol.csv
   分析に使うデータ（587名）。これが無いとコードが動かないので必須。

・変数表.xlsx
   変数やカテゴリの意味が書いてある対応表。中身を理解したいとき用の参考資料。

・コード_解説付き/（makeTable2sakana.R / makeTable3sakana.R / makeTable4sakana.R）
   コードの中身をコメントで解説しているバージョン。
   → これはあくまで「自分で中身を理解する用」。
   ※注意：このコメント解説はAIに作らせたものなので、これをそのまま提出すると
     AIで作ったのがモロバレします。理解用としてレポート作るときに参考にしてね

・提出コード/（table2.R / table3.R / table4.R）
   コメントなしの実行用コード。
   → 最終的に提出するのはこの「提出コード」の中のファイルを使うのがおすすめ。このコードは一切解説などつけてないです

【余談】コードの解説とか授業でつけるようにって教えられてるなら，その時は教えてください
　　　　解説（緑色のやつ，コメントアウトって言います）をつけるとAI臭がめっちゃしちゃうんだよね

・結果/（table2_result.csv / table3_result.csv / table4_result.csv）
   コードを実行するとこういう結果ファイルが作れるよ、という見本。
   （実行しなくても、どんな出力になるかを確認できるだけのもの．レポートにはいらないかな）

・レポート/（課題3_分析レポート.docx）
   課題3のレポートを実際にどう書くかの「例」として作ってもらったもの。
   → 一番大事：どこをAIで作って、どこを自分で考えたかを、Table 2・3・4 すべてについて書いてある。提出レポートを書くときは必ずこれを参考にすること！！


■ 実行方法（コードを動かしたいとき）
1. R が入っている環境で、hl_qol.csv と同じ場所に .R ファイルを置く。
2. ターミナルなら  Rscript table2.R  のように実行。RStudio なら開いて全実行でもOK。
3. 実行すると table2_result.csv / table3_result.csv / table4_result.csv が出てくる。

ー＞よくわからなかったら，それぞれのファイルの生テキストをコピーして自分のファイルに上書きで貼り付ければOK！


■ 分析のメモ（中身の補足）
・Table 2 はヘルスリテラシー(hltcatnew)が欠損の人を除外して集計（有効n=574）。
・Table 3 は全587名を使って、各カテゴリの度数(N)も一緒に出している。
・Table 4 は年齢を連続量 age とその2乗項 age2 で入れている（有効n=574）。
・解説付き版と提出コード版は、出てくる結果が完全に同じことを確認済み。
